# sp0t-dl-tg

Python Telegram bot for searching music content, processing Spotify links, and using YouTube Music links as search input. The bot runs with polling, replies in private chats, and can also work in groups when mentioned.

> Responsible use: use this project only with accounts, files, sessions, and content you are authorized to use. Do not commit tokens, cookies, device files, or personal data to the repository.

## What It Does

- Accepts Spotify track, album, and playlist links.
- Provides `/search` for songs, albums, and playlists.
- Detects YouTube Music links and converts them into equivalent search queries.
- Shows Telegram inline buttons for search results.
- Processes songs one by one and sends them to the chat as audio files.
- Adds ID3 metadata to the final file, including title, artist, album, date, copyright, and cover art.
- Cleans temporary files after each operation.

## Project Structure

## Requirements

- Python 3.10 or newer.
- A Telegram account and a bot created with BotFather.
- An account compatible with the workflow used by this project.
- `ffmpeg` available in `PATH`.
- `mp4decrypt` available in `PATH`.
- Local runtime files required by your environment:
  - `cookies.txt`
  - `device.wvd`

This project does not include tokens, cookies, device files, sessions, or credentials. Those files must stay outside version control.

## Installation

1. Clone the repository:

```bash
git clone <repo-url>
cd sp0t-dl-tg
```

2. Create and activate a virtual environment:

```bash
python -m venv venv
```

On Windows PowerShell:

```powershell
.\venv\Scripts\Activate.ps1
```

On macOS/Linux:

```bash
source venv/bin/activate
```

3. Install dependencies:

```bash
pip install -r requirements.txt
```

4. Verify that the external tools are available:

```bash
ffmpeg -version
mp4decrypt --version
```

If either command is missing, install the corresponding tool and add it to `PATH`.

## Configuration

1. Copy the environment template:

```bash
cp .env.example .env
```

On Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

2. Edit `.env` and add your Telegram bot token:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
```

3. Place the required private files in the project root:

```text
cookies.txt
device.wvd
```

Do not commit these files. They are already covered by `.gitignore`.

## Running

With the virtual environment active:

```bash
python script.py
```

If everything is configured correctly, the console will show startup messages and the bot will begin listening for updates through polling.

## Telegram Usage

### Search Content

Send:

```text
/search song or album name
```

The bot will show inline buttons with track, album, and playlist results. Selecting a result starts processing.

### Send a Spotify Link

You can send links such as:

```text
https://open.spotify.com/track/...
https://open.spotify.com/album/...
https://open.spotify.com/playlist/...
```

The bot detects the link type and processes the matching content.

### Send a YouTube Music Link

You can send a `music.youtube.com` link. The bot extracts information from the link with `yt-dlp` and uses it as a Spotify search query.

### Group Usage

In non-private chats, the bot only responds when mentioned. The expected bot username is defined in `script.py` inside `handle_message`:

```python
bot_username = 'sp0tdl_bot'
```

If your bot uses a different username, update that value.

## Important Variables and Paths

These paths are defined in `script.py`:

```python
mp4decrypt_path = "mp4decrypt"
ffmpeg_path = "ffmpeg"
wvd_path = base_dir / "device.wvd"
cookies_path = base_dir / "cookies.txt"
temp_dir = base_dir / "temp"
```

If `ffmpeg` or `mp4decrypt` are not in `PATH`, you can replace those variables with local absolute paths. Avoid committing personal paths if they include usernames or private information.

## Security

Do not commit:

- `.env`
- `cookies.txt`
- `device.wvd`
- logs containing API responses, tokens, or account information
- temporary downloaded or processed files

Recommended practices:

- Use a dedicated bot token for this project.
- Revoke and rotate the token if it is ever exposed.
- Do not share cookies or device files.
- Keep the bot in a private or controlled environment.
- Review the permissions and policies of the platforms you use with the bot.

## Troubleshooting

### `TELEGRAM_BOT_TOKEN not defined`

The `.env` file is missing, the variable name is wrong, or the environment is not loading the configuration.

Check:

```env
TELEGRAM_BOT_TOKEN=your_telegram_bot_token
```

### `ffmpeg` or `mp4decrypt` Is Not Recognized

The tool is not installed or is not available in `PATH`.

Test with:

```bash
ffmpeg -version
mp4decrypt --version
```

### `Account is not premium. Cannot proceed.`

The script checked the account type and stopped because it does not match the requirement expected by the workflow.

### Search Errors or Unexpected Responses

This can happen because of API response changes, rate limits, expired cookies, or content that is unavailable for the account or region.

### Audio Upload Timeouts

The script retries uploads with backoff. If the issue continues, check server connectivity, file size, and current Telegram limits.

## Main Dependencies

- `python-telegram-bot`: Telegram integration.
- `requests`: HTTP requests.
- `yt-dlp`: media extraction and download handling.
- `mutagen`: ID3 tag writing.
- `pywidevine`: device/license handling in the existing workflow.
- `python-dotenv`: loading variables from `.env`.

## Deployment

The bot uses polling, so it does not require a public webhook. It can run on:

- a local computer,
- a VPS,
- a container,
- a service that keeps Python processes alive.

Deployment recommendations:

- Keep `.env`, `cookies.txt`, and `device.wvd` as environment secrets.
- Run the process with a non-admin user.
- Use rotated logs if running 24/7.
- Restart the process automatically with `systemd`, Docker, PM2, or your preferred supervisor.
- Make sure `ffmpeg` and `mp4decrypt` are available in the deployment environment.

## Maintenance

After updating dependencies or moving the project:

```bash
pip install -r requirements.txt --upgrade
python script.py
```

If you change the bot username, remember to update `bot_username` so group mentions keep working.
